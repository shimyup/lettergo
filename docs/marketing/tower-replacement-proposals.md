# 타워 대신 적용할 레벨업 요소 제안 (Build 120 기준)

현재 "타워" (`activityScore.tier` + `towerFloors` + 타워 스킨) 는 Build 101
이전 "슬로우 소셜 펜팔" 시절의 진급 메타포. Letter Go 가 **픽업 중심
보물찾기** 로 전환된 지금, 타워의 "층이 쌓인다" 은유가 더 이상 게임 루프
와 직결되지 않는다.

이 문서는 타워를 **완전 교체 / 부분 교체 / 보완** 할 수 있는 5가지 후보를
제시하고, 각각의 구현 비용·효과·리스크·기존 자산 활용도를 비교한다.

---

## 현재 타워의 역할 정리 (교체 대상 범위)

| 기능 | 현재 구현 | 교체 필요 여부 |
|------|-----------|----------------|
| 지도 위 내 위치 마커 | `_MyTowerMarker` (건물 모양 + 층 수 표시) | 교체 or 단순화 |
| 지도 위 다른 유저 마커 | `_buildMapTowerMarkers` (클러스터링) | 유지 (사회적 표식) |
| 티어 뱃지 | `activityScore.tier` (beginner/casual/regular/experienced) | 교체 or 병행 |
| 타워 높이 | `towerFloors` (활동 점수 기반 정수) | 의미 재정의 |
| Premium 스킨 | `premiumFeature4` "타워 커스텀 색상" | 다른 커스텀으로 이동 |
| 프로필 "타워 진행바" | `_buildTowerProgressCard` | 교체 or 병행 |

**핵심 결정**: 지도 "다른 유저 존재" 표시는 사회적 fabric 으로 유지하되,
**내 레벨·진급 비주얼은 픽업 메타포로 전환** 하는 것이 헌트 포지셔닝과
정합.

---

## 🥇 후보 1 — **픽업 반경 링 + 헌터 아바타** (주인공 승격)

### 컨셉
Build 120 에서 추가한 픽업 반경 링 자체가 **레벨의 시각적 결과물**. 링
중앙엔 작은 원형 헌터 아바타 (🗺/🎯/🧭 이모지 + 국가 플래그). 링 크기 ·
색 = 레벨 · 티어. 타워 마커 전면 교체.

### 시각
```
       ╭─────────────╮
       │             │
       │      🎯     │   ← 내 아바타 (작게)
       │   🇰🇷        │
       │             │
       ╰─────────────╯   ← 반경 링 (확대 = 레벨업)
```

### 레벨업 → 링 확장 애니메이션
레벨업 시 200–400ms ease-out 으로 링이 +10m 만큼 확대. 동시에 링 색이
한 번 깜박 (alpha 0.8 → 0.3). **직접 체감**.

### 프로필
"내 타워" 섹션 → "내 헌트 프로필". 중앙에 큰 원형 프리뷰 (사용자 플래그 +
레벨 숫자 + 현재 반경). 하단에 HuntWalletCard 의 반경 진행바 재사용.

### 구현 비용 · 기존 자산 활용
- ✅ 반경 링 이미 존재 (Build 120)
- ✅ 아바타용 국가 플래그 이미 있음
- ⏱ `_MyTowerMarker` 교체: 건물 → 원형 아바타 (~80 LOC)
- ⏱ 프로필 `_buildTowerProgressCard` → `_buildHunterProfileCard`
- ❌ 다른 유저 타워 마커는 유지 (사회적 표식 용도)
- ❌ Premium "타워 커스텀 색상" → "헌터 아바타 배경" 으로 리포지션
- **총 비용**: Medium (2–3 빌드)

### Pros
- 이미 핵심 시각(링) 이 기존에 있어 재사용 극대
- "레벨업 = 반경 확대" 일체화. 가장 강력한 게임플레이 ↔ 시각 정합
- 타워라는 정적 은유 대신 **유저 본인 중심** 의 동적 시각

### Cons
- 지도에서 타워가 사라지면 심미적 아이덴티티 일부 손실 (기존 빌드 사진
  썸네일 재촬영 필요)
- 다른 유저 타워 마커는 유지하면서 내 것만 교체 → 마커 유형 혼재

## 🥈 후보 2 — **헌터 랭크 뱃지 + 스탬프 앨범**

### 컨셉
타워 층 대신 **스탬프 앨범** 이 레벨의 결과물. 각 레벨마다 획득 스탬프.
지도 위는 컴팩트한 랭크 뱃지 (🥉/🥈/🥇/💎/👑). 프로필엔 스탬프 그리드.

### 레벨 → 뱃지 매핑 예시
| 레벨 | 뱃지 | 라벨 |
|------|------|------|
| 1–4 | 🥉 | 견습 헌터 |
| 5–9 | 🥈 | 초보 헌터 |
| 10–19 | 🥇 | 숙련 헌터 |
| 20–34 | 💎 | 전문 헌터 |
| 35–50 | 👑 | 전설의 헌터 |

### 지도 마커
작은 원형 뱃지 (32×32) + 국가 플래그. 현재 타워보다 훨씬 컴팩트.

### 프로필 "스탬프 앨범"
5×10 그리드 (50개 슬롯). 현재 레벨까지 채워진 스탬프 + 향후 획득할 자리.
Premium = 스탬프 테두리 골드.

### 구현 비용
- ⏱ 뱃지 에셋 (이모지 활용 시 에셋 불요)
- ⏱ `_MyTowerMarker` 교체 (~50 LOC)
- ⏱ 스탬프 앨범 신규 위젯 (~150 LOC)
- **총 비용**: Medium (2 빌드)

### Pros
- "보물찾기 = 수집" 메타포 직관적 정합
- Pokémon GO · Geocaching 류 기존 제품들 검증된 수집 동기 차용
- 프로필이 더 "성취 진열장" 처럼 보임

### Cons
- 새로운 시각 언어 만들어야 함 (뱃지 디자인 통일성 필요)
- 반경 링과 뱃지 두 축이 공존 → 프라이머리 지표 혼선 가능

## 🥉 후보 3 — **헌터 백팩**

### 컨셉
지도 마커 = **배낭을 멘 작은 캐릭터**. 백팩 크기/색 = 레벨. 픽업할 때마다
백팩이 잠시 반짝. 프로필 = 백팩 인벤토리 (최근 주운 편지 썸네일).

### 시각
🎒 크기가 점점 커지는 애니메이션. 레벨업 시 백팩 폭 +5%.

### Pros
- "줍기" 메타포와 1:1 매핑 (아이템을 줍어 배낭에 넣는다)
- "내 인벤토리" 로서 Hunt Wallet 과 자연스럽게 연결
- 귀여운 시각 — 유저 친화적

### Cons
- 캐릭터 에셋 디자인 필요 (일러스트 또는 이모지 조합 제한적)
- Flutter 에서 세밀한 캐릭터 애니메이션 구현 부담

### 구현 비용
- ⏱ 캐릭터/백팩 에셋 필요 (외부 일러스트레이터 또는 Lottie)
- ⏱ 마커 애니메이션 (~100 LOC)
- **총 비용**: Medium-High (2–4 빌드 + 에셋)

---

## 🕐 후보 4 — **스트릭 불꽃 강화**

### 컨셉
기존 `streak_badge` 를 확장. **연속 픽업 일수** 가 레벨의 주요 지표.
불꽃 색·크기가 스트릭 길이에 비례. 지도 마커는 미니 불꽃 + 플래그.

### Pros
- 이미 `streak_badge.dart` 위젯 존재 (코드 재사용)
- "매일 픽업" 동기 부여 (리텐션 핵심)

### Cons
- 스트릭은 "연속성" 지표 — 레벨의 누적성과 의미 중복
- 하루 빠지면 스트릭 0 → 레벨 시각이 리셋되는 인상 (혼란)

### 구현 비용
- ⏱ 기존 StreakBadge 리팩터 (~60 LOC)
- **총 비용**: Low

## 🕐 후보 5 — **베이스캠프 / 나침반 성장**

### 컨셉
- **베이스캠프**: 타워와 너무 유사 (세로 건물 → 가로 캠프). 인상이 약함.
- **나침반 성장**: 멋있지만 구현 비용 크고 "내가 어디 있는지" 메타포와 약간
  동떨어짐.

### 권장: **배제**

## 🎯 권장

### 주력 방안 — **후보 1 (반경 링 + 헌터 아바타)**

**이유**:
1. Build 120 에서 이미 반경 링을 구현해 핵심 자산 보유
2. 반경 링 = "내가 줍을 수 있는 범위" = 게임 핵심 가치 자체
3. 레벨업 → 링 확장 애니메이션으로 즉각적 체감
4. 타워는 다른 유저 위치용으로만 남기고 점진적 축소
5. Premium 차별화(골드 링 vs 티일 링) 이미 시각 완성

**로드맵**:
- **Build 121**: `_MyTowerMarker` 교체. 건물 → 원형 헌터 아바타 (플래그 + 레벨 숫자). 링은 유지.
- **Build 122**: 레벨업 시 링 확장 애니메이션 추가. 프로필 `_buildTowerProgressCard` → `_buildHunterProfileCard` 리브랜딩.
- **Build 123**: Premium "타워 커스텀" → "헌터 아바타 배경 테마" 로 리포지션.

### 보조 방안 — **후보 2 (스탬프 앨범)** 을 프로필 한정 추가

지도는 후보 1로, 프로필은 기존 타워 대신 스탬프 앨범 을 도입 가능.
"마일스톤 레벨(2/5/10/25/50) 에서만 스탬프 획득" 식의 희소성 모델.
Build 124+ 에서 고려.

### 배제 — **후보 3 (백팩)**, **후보 4 (스트릭)**, **후보 5 (캠프/나침반)**

- 백팩: 에셋 비용 대비 차별화 미미
- 스트릭: 레벨과 의미 중복
- 캠프/나침반: 구현 비용 · 메타포 불명확

---

## ⚠️ 주의 사항

- 다른 유저 타워 마커 (사회적 fabric) 는 **유지**. 내 마커만 교체 필요.
- 기존 Premium 가입자에게 "타워 스킨" 이라는 약속을 깨지 않도록,
  후속 리포지셔닝 시 "헌터 아바타 배경" 으로 이름만 변경하고 자동 이관.
- 타워 관련 l10n 키(`towerFloors`, `mapTowerLabel` 등) 는 새 용어로 리라이팅
  또는 병행 노출. 14개 언어 × ~10개 키 = ~140 문자열 분량 재번역 필요.
- 스토어 스크린샷: 타워가 보이는 컷 전부 재촬영. Build 120 QA 체크리스트
  포함.
