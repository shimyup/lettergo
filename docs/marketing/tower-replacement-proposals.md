# 타워 대신 적용할 레벨업 요소 제안 (Build 121 업데이트)

> **Build 121 결정 반영**: 주력 방안(후보 1 — 픽업 반경 링 + 헌터 아바타)
> 과 보조 방안(후보 2 — 스탬프 앨범 ▸ **아이템형 이모지 스탬프** 로 변경)
> 을 동시에 구현 완료. 아래는 결정 맥락 + 구현된 내용을 정리한 문서.

현재 "타워" (`activityScore.tier` + `towerFloors` + 타워 스킨) 는 Build 101
이전 "슬로우 소셜 펜팔" 시절의 진급 메타포. Letter Go 가 **픽업 중심
보물찾기** 로 전환된 지금, 타워의 "층이 쌓인다" 은유가 더 이상 게임 루프
와 직결되지 않는다.

이 문서는 타워를 **완전 교체 / 부분 교체 / 보완** 할 수 있는 5가지 후보를
제시하고, 각각의 구현 비용·효과·리스크·기존 자산 활용도를 비교한다.

---

## 현재 타워의 역할 정리 (교체 대상 범위)

| 기능 | 현재 구현 | 교체 필요 여부 |
|------|-----------|----------------|
| 지도 위 내 위치 마커 | `_MyTowerMarker` (건물 모양 + 층 수 표시) | 교체 or 단순화 |
| 지도 위 다른 유저 마커 | `_buildMapTowerMarkers` (클러스터링) | 유지 (사회적 표식) |
| 티어 뱃지 | `activityScore.tier` (beginner/casual/regular/experienced) | 교체 or 병행 |
| 타워 높이 | `towerFloors` (활동 점수 기반 정수) | 의미 재정의 |
| Premium 스킨 | `premiumFeature4` "타워 커스텀 색상" | 다른 커스텀으로 이동 |
| 프로필 "타워 진행바" | `_buildTowerProgressCard` | 교체 or 병행 |

**핵심 결정**: 지도 "다른 유저 존재" 표시는 사회적 fabric 으로 유지하되,
**내 레벨·진급 비주얼은 픽업 메타포로 전환** 하는 것이 헌트 포지셔닝과
정합.

## 🥇 후보 1 — **픽업 반경 링 + 헌터 아바타** (주인공 승격)

### 컨셉
Build 120 에서 추가한 픽업 반경 링 자체가 **레벨의 시각적 결과물**. 링
중앙엔 작은 원형 헌터 아바타 (🗺/🎯/🧭 이모지 + 국가 플래그). 링 크기 ·
색 = 레벨 · 티어. 타워 마커 전면 교체.

### 시각
```
       ╭─────────────╮
       │             │
       │      🎯     │   ← 내 아바타 (작게)
       │   🇰🇷        │
       │             │
       ╰─────────────╯   ← 반경 링 (확대 = 레벨업)
```

### 레벨업 → 링 확장 애니메이션
레벨업 시 200–400ms ease-out 으로 링이 +10m 만큼 확대. 동시에 링 색이
한 번 깜박 (alpha 0.8 → 0.3). **직접 체감**.

### 프로필
"내 타워" 섹션 → "내 헌트 프로필". 중앙에 큰 원형 프리뷰 (사용자 플래그 +
레벨 숫자 + 현재 반경). 하단에 HuntWalletCard 의 반경 진행바 재사용.

### 구현 비용 · 기존 자산 활용
- ✅ 반경 링 이미 존재 (Build 120)
- ✅ 아바타용 국가 플래그 이미 있음
- ⏱ `_MyTowerMarker` 교체: 건물 → 원형 아바타 (~80 LOC)
- ⏱ 프로필 `_buildTowerProgressCard` → `_buildHunterProfileCard`
- ❌ 다른 유저 타워 마커는 유지 (사회적 표식 용도)
- ❌ Premium "타워 커스텀 색상" → "헌터 아바타 배경" 으로 리포지션
- **총 비용**: Medium (2–3 빌드)

### Pros
- 이미 핵심 시각(링) 이 기존에 있어 재사용 극대
- "레벨업 = 반경 확대" 일체화. 가장 강력한 게임플레이 ↔ 시각 정합
- 타워라는 정적 은유 대신 **유저 본인 중심** 의 동적 시각

### Cons
- 지도에서 타워가 사라지면 심미적 아이덴티티 일부 손실 (기존 빌드 사진
  썸네일 재촬영 필요)
- 다른 유저 타워 마커는 유지하면서 내 것만 교체 → 마커 유형 혼재

## 🥈 후보 2 — **아이템형 이모지 스탬프** ✅ Build 121 구현

### 최종 컨셉 (변경됨)
스탬프 앨범 그리드(50칸)는 "도감 채우기" 느낌이 강해 **헌트 본질(주워 쓰는
혜택)** 과 거리 있음. 대신 "주워 모은 **장비**" 인벤토리 형식으로 전환.
마일스톤 레벨 5개(Lv 2 / 5 / 10 / 25 / 50) 에 각각 **대표 아이템 이모지**
를 매핑해 헌터의 장비함처럼 노출.

### 아이템 매핑
| 레벨 | 이모지 | 의미 |
|------|--------|------|
| 2 | 🎯 | 조준 — 첫 마일스톤 |
| 5 | 🧭 | 나침반 |
| 10 | 🗺 | 보물 지도 |
| 25 | 🎒 | 여행자 배낭 |
| 50 | 👑 | 전설의 왕관 |

### 시각 (구현 완료)
- **프로필 HuntWalletCard** 반경 바 바로 아래 "헌터 아이템" 섹션 (5칸 가로
  슬롯). 획득 = 풀컬러 이모지 + 골드 테두리 + `Lv N`, 미획득 = 회색 이모지
  + 🔒 + 툴팁 "Lv N 에 해금".
- **지도 내 아바타 좌상단** 에 가장 높이 획득한 아이템 이모지 작게 (12px).
  "지금까지 주운 대표 장비" 를 한 눈에.

### 구현 위치
- `AppState`: `_hunterItemEmojis` Map + `hunterItemEmoji(level)` +
  `earnedHunterItemLevels` + `latestHunterItemEmoji` getter
- `HuntWalletCard._buildHunterItems()` — 프로필 5-슬롯 줄
- `_HunterItemSlot` (신규 위젯) — 개별 슬롯
- `_MyTowerMarker` → `milestoneItemEmoji` 파라미터로 아바타 좌상단 노출

### 왜 "아이템" 인가 (앨범 그리드 대비)
- **수량 적음** — 5개 vs 50개. 과도한 수집 압박 없음
- **의미 명확** — 각 이모지가 도구/장비로 읽혀 "줍기" 메타포 직결
- **지도와 연동** — 가장 높은 아이템을 내 아바타 뱃지로 재사용 가능
- **Premium 충돌 없음** — 이미 골드 링·커스텀 배경으로 차별화 있어
  스탬프 테두리 같은 추가 장치 불필요
- **구현 비용 ↓** — 이모지만 쓰므로 디자인 에셋 제작 불요

## 🥉 후보 3 — **헌터 백팩**

### 컨셉
지도 마커 = **배낭을 멘 작은 캐릭터**. 백팩 크기/색 = 레벨. 픽업할 때마다
백팩이 잠시 반짝. 프로필 = 백팩 인벤토리 (최근 주운 편지 썸네일).

### 시각
🎒 크기가 점점 커지는 애니메이션. 레벨업 시 백팩 폭 +5%.

### Pros
- "줍기" 메타포와 1:1 매핑 (아이템을 줍어 배낭에 넣는다)
- "내 인벤토리" 로서 Hunt Wallet 과 자연스럽게 연결
- 귀여운 시각 — 유저 친화적

### Cons
- 캐릭터 에셋 디자인 필요 (일러스트 또는 이모지 조합 제한적)
- Flutter 에서 세밀한 캐릭터 애니메이션 구현 부담

### 구현 비용
- ⏱ 캐릭터/백팩 에셋 필요 (외부 일러스트레이터 또는 Lottie)
- ⏱ 마커 애니메이션 (~100 LOC)
- **총 비용**: Medium-High (2–4 빌드 + 에셋)

## 🕐 후보 4 — **스트릭 불꽃 강화**

### 컨셉
기존 `streak_badge` 를 확장. **연속 픽업 일수** 가 레벨의 주요 지표.
불꽃 색·크기가 스트릭 길이에 비례. 지도 마커는 미니 불꽃 + 플래그.

### Pros
- 이미 `streak_badge.dart` 위젯 존재 (코드 재사용)
- "매일 픽업" 동기 부여 (리텐션 핵심)

### Cons
- 스트릭은 "연속성" 지표 — 레벨의 누적성과 의미 중복
- 하루 빠지면 스트릭 0 → 레벨 시각이 리셋되는 인상 (혼란)

### 구현 비용
- ⏱ 기존 StreakBadge 리팩터 (~60 LOC)
- **총 비용**: Low

## 🎯 구현된 최종 방안 (Build 121)

### 주력 — **후보 1 (반경 링 + 헌터 아바타)** ✅

**적용**:
- `_MyTowerMarker` 내부 시각 전면 교체: 사각 건물 + 층 수 → **원형 아바타
  + 국가 플래그 + 중앙 레벨 뱃지**
- 색상: Brand 오렌지 · Premium 골드 · Free 티일 (Build 120 픽업 반경 링
  과 완전 동일)
- 맥동 링 + Glow 그대로 유지 → 지도에서 내 위치는 여전히 발견하기 쉬움
- `pendingLetterCount` 뱃지 유지 (수령 대기 편지 N개 오렌지 뱃지)
- 클래스 이름은 하위 호환 위해 `_MyTowerMarker` 그대로, 내부만 재작성

### 보조 — **후보 2 (아이템형 이모지 스탬프)** ✅

**적용**:
- `AppState._hunterItemEmojis`: Lv 2 🎯 / 5 🧭 / 10 🗺 / 25 🎒 / 50 👑
- `earnedHunterItemLevels` getter: 현재 레벨이 마일스톤 이상이면 소유
- `latestHunterItemEmoji`: 가장 높이 획득한 아이템 → 지도 아바타 좌상단
- `HuntWalletCard._buildHunterItems()`: 프로필 5-슬롯 가로 줄, 획득/잠김
  시각 구분
- `_HunterItemSlot`: 개별 슬롯 위젯 (풀컬러 vs 회색 + 🔒)

### 후속 (선택사항 — Build 122+)

- **레벨업 시 링 확장 애니메이션** (200–400 ms ease-out + alpha 펄스)
- **프로필 `_buildTowerProgressCard` → `_buildHunterProfileCard`** 명칭·
  카피 리브랜딩 (내부 로직은 그대로)
- **Premium "타워 커스텀 색상" → "헌터 아바타 배경 테마"** 로 리포지션
  (기존 구독자 자동 이관)

### 배제 유지

- **후보 3 백팩** — 후보 2(아이템형 이모지) 가 같은 "주워 모은 장비" 메타포
  를 에셋 없이 커버
- **후보 4 스트릭 불꽃** — 레벨과 의미 중복
- **후보 5 캠프/나침반** — 구현 비용 큰 반면 메타포 불명확

## ⚠️ 주의 사항 (Build 121 반영 상태)

### 이번 빌드에 적용된 정책

- ✅ **다른 유저 타워 마커는 유지** — 사회적 fabric. `_buildMapTowerMarkers`
  및 `_TowerClusterMarker` 경로 손대지 않음.
- ✅ **`_MyTowerMarker` 클래스 이름 보존** — 호출부 하위 호환. 내부 렌더링만
  재작성해 회귀 리스크 최소화.
- ✅ **헌터 아이템은 Brand 계정에서 숨김** — Brand 는 XP/레벨 시스템 밖이라
  아이템 슬롯이 의미 없음. `state.currentUser.isBrand` 로 게이트.
- ✅ **Premium "타워 커스텀 색상" 약속 자동 반영** — 아바타 본체 테두리
  색이 Premium → 골드 로 자동 적용됨. 별도 이관 모달 불필요.

### 다음 빌드에서 처리

- 🕐 **타워 관련 l10n 키 병행 노출** — `towerFloors`, `mapTowerLabel` 등
  기존 키는 남겨두되, 새 "헌터 아바타 / 헌터 아이템" 맥락 키 (`hunterItemsTitle`,
  `hunterItemLockedHint` 이미 Build 121 추가) 로 점차 전환. 기존 타워
  라벨이 불필요해진 화면부터 순차 제거.
- 🕐 **프로필 `_buildTowerProgressCard` 리브랜딩** — 내부 로직 유지, 명칭과
  헤더 카피만 "내 타워" → "내 헌터 프로필" 로 (Build 122 계획).
- 🕐 **스토어 스크린샷 재촬영 필요** — 내 마커가 건물에서 원형 아바타로
  바뀌므로 메인 지도 스크린샷 전부 업데이트. Build 120 QA 체크리스트와
  함께 관리.
- 🕐 **레벨업 링 확장 애니메이션** — 현재는 링이 즉시 확대. 부드럽게
  보이도록 `AnimationController` 기반 tween 추가는 Build 122 후보.

### 모니터링할 리스크

- **지도 아바타가 너무 작아 보이는지** — Build 121 아바타는 46 × 46.
  기존 타워(44 × 54) 와 비슷한 체적이지만 원형이라 체감 크기 더 작을 수
  있음. 실기기 QA 후 조정 가능.
- **플래그 가독성** — 24 px 이모지 플래그가 저사양 기기에서 블러 처리될
  수 있음. Canvas 렌더링 품질 확인 필요.
- **헌터 아이템 Lv 50 오래 걸림** — 현재 마일스톤은 2/5/10/25/50 으로
  Lv 25–50 사이 공백이 큼. 중간에 Lv 40 을 추가하거나 하는 조정은 실데이터
  리텐션 커브 본 후.
